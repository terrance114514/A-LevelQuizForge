# 小程序生成提示词（可直接复制给代码生成模型）

## Prompt 1: 生成 A-Level 学习小程序基础版

你是一个资深全栈工程师，请生成一个“国际高中 A-Level 学习助手”小程序（微信小程序或 UniApp 均可）。

目标用户：国际高中生（G1/G2/AS/A2）。

核心功能：
1. 首页支持选择年级（G1/G2/AS/A2）、考试局（CIE/Edexcel/AQA/OCR）、科目（Math/Physics/Chemistry/Biology/Economics/CS）
2. 若选择 CIE + Mathematics，支持细分卷别：P1/P2/P3/P4/S1/S2/M1
3. 点击“生成模拟试卷”进入新页面，可选择：题目数量、难度（基础/中等/冲刺）、知识点（多选）
4. 生成题目列表后可作答并提交
5. 提交后进入“错题分析”页面，展示：
   - 按知识点错误率柱状图
   - 高频错误类型（审题失误/计算错误/概念不清）
   - 学习建议（短期7天、长期30天）

技术要求：
- 提供完整目录结构
- 提供至少3个页面（首页、组卷页、分析页）
- 提供 mock 数据和 API 占位
- 代码包含中文注释
- 样式简洁、适合学生使用
- 页面跳转与参数传递要完整可运行

请按“文件树 + 各文件代码”的格式输出。

## Prompt 2: 升级为可扩展的后端驱动版本

在已有小程序基础上，升级为“前后端分离”架构：

后端能力：
1. 用户系统（学生、教师）
2. 题库管理（考试局/科目/模块/知识点/难度标签）
3. 智能组卷接口（按题量、难度比例、知识点覆盖）
4. 错题记录与错因分类
5. 学习建议生成（基于近10次练习数据）

技术要求：
- 后端使用 Node.js + Express 或 Python FastAPI
- 数据库使用 MySQL 或 PostgreSQL
- 提供 RESTful API 文档示例
- 对每个 API 给出请求/响应 JSON 示例
- 说明如何部署到云服务器

请输出：
1. 数据库表设计
2. API 列表
3. 关键后端代码
4. 小程序端调用示例

## Prompt 3: 教研级个性化推荐

请继续增强系统，增加“个性化学习路径推荐”：
- 输入：学生最近练习记录、知识点掌握率、考试目标分数
- 输出：7天和30天学习计划、每日建议题量、优先复习知识点
- 给出可解释原因（为什么推荐这个知识点）
- 提供提醒机制（连续3天未练习提醒）

请提供：
- 推荐算法思路（规则+简单模型）
- 伪代码
- 前端展示样式方案
- A/B测试指标（正确率提升、完成率、留存）
